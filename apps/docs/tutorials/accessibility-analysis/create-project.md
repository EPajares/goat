Every analysis in GOAT starts with a project. Projects help you organize your work and collaborate with team members.

## Step-by-Step Instructions

### 1. Navigate to the Workspace

After logging in, you'll see the GOAT workspace. This is your central hub for all projects and datasets.

<div style={{ display: 'flex', flexDirection: 'column', alignItems: 'center' }}>

  <p><em>The GOAT Workspace homepage</em></p>
</div>

### 2. Create a New Project

1. Click the **"New Project"** button in the top right corner
2. Give your project a descriptive name (e.g., "Munich Accessibility Analysis")
3. Optionally add a description and tags
4. Select a geographic area for your project

<div style={{ display: 'flex', flexDirection: 'column', alignItems: 'center' }}>
  <img src={require('/img/workspace/workspace_projects.png').default} alt="GOAT Projects" style={{ maxHeight: "500px", maxWidth: "100%", objectFit: "cover", borderRadius: "8px", boxShadow: "0 4px 12px rgba(0,0,0,0.15)"}}/>
  <p><em>Your projects list</em></p>
</div>

### 3. Configure Project Settings

In the project settings, you can:
- Set the default map extent
- Configure coordinate reference systems
- Add collaborators to your project
- Set project visibility (private/shared)

## Tips for Project Organization

:::tip Best Practice
Use a consistent naming convention for your projects. For example: `[City]_[Analysis Type]_[Date]` like "Munich_Catchments_2026"
:::

:::info 
Projects can contain multiple layers and analyses. Think of a project as a complete study or report.
:::
